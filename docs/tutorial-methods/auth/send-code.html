sendCode | ارسال کد تایید

این متد تابعی از کلاس Client است که کد تایید را به شماره تلفن مورد نظر ارسال می‌کند.

توضیح کلی​

متد sendCode یک درخواست رمزنگاری‌شده به سرور ارسال می‌کند تا کد تایید به شماره تلفن داده‌شده ارسال شود. این متد از متد عمومی builder استفاده می‌کند و می‌تواند نوع ارسال کد را به صورت "SMS" یا "Internal" تعیین کند.

ورودی‌ها​

نام پارامترنوعتوضیح
phone_numberstringشماره تلفنی که کد تایید باید به آن ارسال شود.
pass_keystring یا nullکلید عبور موقت یا رمز (اختیاری، پیش‌فرض null).
send_typestringنوع ارسال کد، که می‌تواند "SMS" یا "Internal" باشد. پیش‌فرض "SMS".

خروجی​

در صورت موفقیت، این متد یک Promise از نوع SendCode بازمی‌گرداند.

interface SendCode {
  phone_code_hash: string;
  status: string;
  code_digits_count: number;
  has_confirmed_recovery_email: boolean;
  no_recovery_alert: string;
  send_type: string;
  hint_pass_key: string;
}

🔗 برای جزئیات بیشتر به صفحه پاسخ‌های مربوط به احراز هویت مراجعه کنید.

نکات مهم​

  • اگر مقدار send_type غیر از "SMS" یا "Internal" باشد، به صورت پیش‌فرض "SMS" در نظر گرفته می‌شود و هشدار در کنسول نمایش داده می‌شود.

  • این متد به صورت خودکار از سشن موقت (tmp_session = true) استفاده می‌کند.
