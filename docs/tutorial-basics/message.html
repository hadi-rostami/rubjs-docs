کلاس Message

کلاسی که برای مدیریت آپدیت‌های پیام (MessageUpdate) در سیستم استفاده می‌شود. این کلاس متدهایی برای تعامل با پیام مثل ارسال جواب، پین کردن، فوروارد کردن، حذف و واکنش دارد.

تابع کمکی​

getOriginalType(message)

  • ورودی: یک آبجکت پیام MessageUpdate.
  • خروجی: نوع اصلی پیام (original type).

    منطق:

  • اگر پیام یا نوع آن موجود نباشد، مقدار "Delete" برمی‌گرداند.
  • اگر نوع پیام شامل "FileInline" باشد، نوع فایل را از file_inline.type استخراج می‌کند.
  • اگر نوع پیام "Event" باشد، نوع رویداد را از event_data.type برمی‌گرداند.
  • در غیر اینصورت، همان message.type را برمی‌گرداند.

سازنده (constructor)​

  • مقداردهی اولیه تمام خصوصیات از update.
  • افزودن ویژگی‌های client و originalType به صورت غیر قابل شمارش (enumerable: false) با استفاده از Object.defineProperty.

متدها​

reply​

async reply(
  text?: string,
  object_guid?: string,
  message_id?: string,
  auto_delete?: number,
  file_inline?: string | Buffer<ArrayBufferLike>,
  type?: string,
  is_spoil?: boolean,
  thumb?: string,
  audio_info?: boolean
) {
  return await this.client.sendMessage(
    object_guid || this.object_guid,
    text,
    message_id || this.message_id,
    file_inline,
    type,
    is_spoil,
    thumb,
    audio_info,
    auto_delete
  );
}
  • ارسال پیام جواب به پیام فعلی.
  • پارامترها برای متن، شناسه شیء، شناسه پیام، خودکار حذف شدن، فایل ضمیمه، نوع، وضعیت - اسپویلر، تصویر بندانگشتی و اطلاعات صوتی هستند.
  • استفاده از متد sendMessage کلاینت.

pin​

async pin(
  object_guid?: string,
  message_id?: string,
  action: "Pin" | "Unpin" = "Pin"
) {
  return await this.client.setPinMessage(
    object_guid || this.object_guid,
    message_id || this.message_id,
    action
  );
}
  • پین یا آنپین کردن پیام.
  • پارامترها: شناسه شیء، شناسه پیام و نوع اکشن (پین یا آنپین).
  • استفاده از متد setPinMessage کلاینت.

reply_photo​

async reply_photo(
    photo: string | Buffer<ArrayBufferLike>,
    caption?: string,
    object_guid?: string,
    auto_delete?: number,
    reply_to_message_id?: string,
    is_spoil?: boolean,
    thumb?: string
) {
  return await this.reply(
    caption,
    object_guid,
    reply_to_message_id,
    auto_delete,
    photo,
    "Image",
    is_spoil,
    thumb
  );
}
  • ارسال عکس به عنوان جواب.
  • پارامترها: فایل عکس، کپشن، شناسه شیء، حذف خودکار، شناسه پیام جواب، اسپویلر و تصویر بندانگشتی.
  • این متد در واقع از reply استفاده می‌کند با type = "Image".

forward​

async forward(to_object_guid: string) {
    return await this.client.forwardMessages(this.object_guid, to_object_guid, [
        this.message_id,
    ]);
}
  • فوروارد کردن پیام به یک شناسه‌ی دیگر.
  • استفاده از متد forwardMessages کلاینت.

delete​

async delete() {

  ]);

  • حذف پیام فعلی.
  • استفاده از متد deleteMessages کلاینت.

reaction​

async reaction(
  reaction_id: number,
  object_guid?: string,
  message_id?: string,
  action: "Add" | "Remove" = "Add"
) {
  return await this.client.actionOnMessageReaction(
    object_guid || this.object_guid,
    message_id || this.message_id,
    reaction_id,
    action
  );
}
  • افزودن یا حذف واکنش به پیام.
  • پارامترها: شناسه واکنش، شناسه چت, شناسه پیام و نوع اکشن (اضافه یا حذف).
  • استفاده از متد actionOnMessageReaction کلاینت.

getReplyMessage​

async getReplyMessage(object_guid?: string, message_id?: string) {
  const result = await this.client.getMessagesByID(
    object_guid || this.object_guid,
    message_id || [this.message.reply_to_message_id]
  );
  return result.messages[0];
}
  • دریافت پیام جواب داده شده به پیام فعلی.
  • استفاده از متد getMessagesByID کلاینت.
  • برمی‌گرداند پیام اول از نتیجه.
